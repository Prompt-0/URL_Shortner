## 2024-07-04 - Authorization Bypass in Alternative Endpoints
**Vulnerability:** The API endpoint (`/api/v1/links/{code}`) and Stats endpoint (`/stats/{code}`) were exposing the `original_url` (and analytics data) of password-protected short links to unauthenticated users, while the primary redirect endpoint (`/{code}`) correctly blocked access.
**Learning:** Security checks implemented on the primary endpoint (e.g., redirecting) were not consistently applied across all endpoints that access the same resource. This is a common pattern where alternative interfaces (APIs, analytics, exports) inadvertently bypass the security constraints of the main interface.
**Prevention:** When adding security constraints (like password protection) to a resource, identify and update *all* endpoints that expose data related to that resource, not just the primary consumer. Centralize access control checks closer to the data retrieval layer rather than solely in the handler layer when possible.
